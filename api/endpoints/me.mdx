## Get Current User

<Note>
Requires authentication via Better Auth session (cookie or header).
</Note>

<CodeGroup>
```bash curl
curl -X GET https://jottar.vercel.com/api/me \
  -H "Cookie: better-auth.session_token=your-session-token"
```

```typescript TypeScript
const me = await api<{ data: { id: string; email?: string } }>("/api/me");

if (me.ok) {
  console.log("User:", me.data);
}
```
</CodeGroup>

### Response

Returns the authenticated user object.

<ResponseField name="ok" type="boolean" required>
  Success indicator (always `true` for successful responses)
</ResponseField>

<ResponseField name="data" type="object" required>
  The user object
  
  <Expandable title="User properties">
    <ResponseField name="id" type="string" required>
      User unique identifier (UUID)
    </ResponseField>
    
    <ResponseField name="email" type="string">
      User email address
    </ResponseField>
    
    <ResponseField name="name" type="string">
      User display name
    </ResponseField>
    
    <ResponseField name="createdAt" type="string">
      Account creation timestamp (ISO 8601)
    </ResponseField>
  </Expandable>
</ResponseField>

### Example Response

<Tabs>
  <Tab title="Success">
    ```json
    {
      "ok": true,
    ```

## Create Default Folders

Creates default folders for the authenticated user if they don't already exist. This endpoint is idempotent.

<CodeGroup>
```bash curl
curl -X POST https://jottar.vercel.com/api/me/default-folders \
  -H "Cookie: better-auth.session_token=your-session-token"
```

```typescript TypeScript
const result = await api<{ ok: boolean }>("/api/me/default-folders", {
  method: "POST",
});

if (result.ok) {
  console.log("Default folders created");
}
```
</CodeGroup>

### Default Folders Created

This endpoint creates two default folders:

1. **Shared Notes** (`slug: "shared-notes"`)
   - Description: "Notes you made available to others"
   
2. **Imported Notes** (`slug: "imported-notes"`)
   - Description: "Notes you added from other people's shared notes"

<Note>
If the folders already exist, this endpoint does nothing and returns successfully. It's safe to call multiple times.
</Note>

### Response

  Success indicator (always `true` for successful responses)
</ResponseField>
