## Introduction

The Jottar API is a RESTful API that allows you to manage notes, folders, and tags programmatically. It's designed for both web and mobile applications, with built-in support for React Native.

## Base URL

```
https://jottar.vercel.com
```

For local development:
```
http://localhost:3000
```

## Response Format

All API endpoints use a consistent JSON envelope pattern:

### Success Response
```json
{
  "ok": true,
  "data": { /* resource data */ }
}
```

### Error Response
```json
{
  "ok": false,
  "error": "Error message"
}
```

<Note>
The `ok` field is always present and indicates whether the request was successful. Use this field to determine how to handle the response in your application.
</Note>

## Content-Type Requirements

All requests with a body (POST, PATCH) must include:

```
Content-Type: application/json
```

## Authentication

Most API endpoints require authentication using Better Auth sessions. See the [Authentication](/api/authentication) page for details on:

- Cookie-based auth for web applications
- Header-based auth for React Native
- Handling 401 errors

<Warning>
Unauthenticated requests to protected endpoints will return `401` status with `{"ok": false, "error": "Not authenticated"}`.
</Warning>

## Error Handling

The API uses standard HTTP status codes:

<ResponseField name="200" type="Success">
  Request completed successfully
</ResponseField>

<ResponseField name="201" type="Created">
  Resource created successfully
</ResponseField>

<ResponseField name="400" type="Bad Request">
  Invalid request parameters or validation failure
</ResponseField>

<ResponseField name="401" type="Unauthorized">
  Not authenticated or invalid session
</ResponseField>

<ResponseField name="404" type="Not Found">
  Resource not found or ownership check failed
</ResponseField>

### Error Response Example

```json
{
  "ok": false,
  "error": "content is required"
}
```

## Rate Limits

Currently, there are no enforced rate limits on the API. However, please use the API responsibly and avoid making excessive requests.

## Quick Start

### Basic Request Example

```typescript
const API_BASE = "https://jottar.vercel.com";

type ApiOk<T> = { ok: true } & T;
type ApiErr = { ok: false; error: string };
type ApiResponse<T> = ApiOk<T> | ApiErr;

async function api<T>(
  path: string,
  init?: RequestInit,
): Promise<ApiResponse<T>> {
  const res = await fetch(`${API_BASE}${path}`, {
    ...init,
    headers: {
      "Content-Type": "application/json",
      ...(init?.headers ?? {}),
    },
  });

  return res.json();
}
```

### Usage Example

```typescript
// Get current user
const me = await api<{ data: { id: string; email?: string } }>("/api/me");

if (!me.ok) {
  console.error("API error:", me.error);
} else {
  console.log("User:", me.data);
}

// Create a note
const note = await api<{ data: unknown }>("/api/notes", {
  method: "POST",
  body: JSON.stringify({ content: "My note content" }),
});
```

<Info>
For React Native applications, you'll need to explicitly pass authentication headers instead of relying on cookies. See the [Authentication](/api/authentication) page for details.
</Info>
