## Overview

Notes are the core entity in Jottar. Each note contains content stored as a string, with an optional title that can be extracted or manually set. Notes support rich text formatting, organization into folders, and metadata for organizational features.

## Creating notes

Notes are created with a content field that stores the rich text content. The title is optional and can be set separately from the content.

<Steps>
  <Step title="Create a note via API">
    Send a POST request to `/api/notes` with the note content:

    ```typescript
    // POST /api/notes
    {
      "content": "Your note content here"
    }
    ```

    The API validates that content is provided and creates the note for the authenticated user.
  </Step>

  <Step title="Server action alternative">
    You can also create notes using the `createNote` server action:

    ```typescript
    import { createNote } from "@/lib/actions/note-actions";

    const result = await createNote("Your note content");
    if (result.error) {
      console.error(result.error);
    } else {
      console.log("Created note:", result.data);
    }
    ```
  </Step>
</Steps>

<Note>
  The `content` field is required when creating a note. The title is optional and defaults to null in the database.
</Note>

## Rich text editor

Jottar uses **Tiptap** for rich text editing, providing a powerful and extensible editor experience.

### Editor features

The editor toolbar includes the following formatting options:

- **Text formatting**: Bold, italic, strikethrough
- **Lists**: Bullet lists and ordered lists
- **Code**: Inline code and code blocks
- **Structure**: Blockquotes, horizontal rules, hard breaks
- **History**: Undo and redo functionality

</Tip>

## Updating notes

Jottar separates content updates from metadata updates to optimize performance.

### Updating content

To update note content, send a PATCH request with only the content field:

```typescript
// PATCH /api/notes/[id]
{
  "content": "Updated note content"
}
```

### Updating note details

To update the title or folder, send a PATCH request with those fields:

```typescript
// PATCH /api/notes/[id]
{
  "title": "My Note Title",
  "folderId": "folder-uuid-here"
}
```

<Warning>
  You cannot update content and metadata (title/folderId) in the same request. The API will return an error: "Provide either content updates or title/folderId updates in one request."
</Warning>

The separation logic is implemented in `/workspace/source/src/app/api/notes/[id]/route.ts:64-68`:

```typescript
if (hasContent && hasDetails) {
  return apiError(
    "Provide either content updates or title/folderId updates in one request."
  );
}
```

## Note metadata

Notes include several boolean and timestamp fields for organization:

### Boolean flags

- **isPinned**: Pin important notes to the top (default: `false`)
- **favorite**: Mark notes as favorites (default: `false`)
- **archived**: Archive notes to hide from main view (default: `false`)

### Timestamps

- **createdAt**: Automatically set when note is created
- **updatedAt**: Automatically updated on any change
- **trashedAt**: Set to current timestamp when trashed, `null` when active

## Duplicating notes

You can duplicate a note to create a copy with the same content, folder, and tags.

```typescript
// POST /api/notes/[id]/duplicate
```

The duplication process:

1. Finds the original note including its tags
2. Creates a new note with the same content and folder
3. Copies all associated tags via the `noteTags` relation
4. Appends " (Copy)" to the title (or uses "Untitled (Copy)" if no title)
5. Truncates the base title to 248 characters to fit the 255-character limit

Implementation at `/workspace/source/src/app/api/notes/[id]/duplicate/route.ts:19-28`:

```typescript
const baseTitle = note.title ?? "Untitled";
const truncatedTitle = baseTitle.substring(0, 248);
const newTitle = `${truncatedTitle} (Copy)`;

const data = await prisma.note.create({
  data: {
    userId: user.id,
    content: note.content,
    folderId: note.folderId,
    title: newTitle,
    noteTags: {
      create: note.noteTags.map((nt) => ({
        tagId: nt.tagId,
      })),
    },
  },
});
```

## Querying notes

The GET `/api/notes` endpoint supports multiple query modes:

### List mode (default)

Returns all notes for the authenticated user:

```typescript
// GET /api/notes
// or
// GET /api/notes?mode=list
```

### Single note mode

Fetch a specific note by ID:

```typescript
// GET /api/notes?mode=one&id=note-uuid
```

### Overview mode

Returns up to 3 notes for the dashboard: up to 2 pinned notes, then fills remaining slots with unpinned notes:

```typescript
// GET /api/notes?mode=overview
```

Implementation at `/workspace/source/src/app/api/notes/route.ts:50-76`:

```typescript
const pinned = await prisma.note.findMany({
  where: {
    userId: user.id,
    archived: false,
    trashedAt: null,
    isPinned: true,
  },
  orderBy: { updatedAt: "desc" },
  take: 2,
});

const remaining = 3 - pinned.length;

const unpinned = await prisma.note.findMany({
  where: {
    userId: user.id,
    archived: false,
    trashedAt: null,
    isPinned: false,
  },
  orderBy: { updatedAt: "desc" },
  take: remaining,
});

return apiSuccess({ data: [...pinned, ...unpinned] });
```

<Note>
  Overview mode prioritizes pinned notes and excludes archived and trashed notes to show the most relevant recent content.
</Note>
