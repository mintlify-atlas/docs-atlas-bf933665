## Overview

Jottar uses [Better Auth](https://www.better-auth.com/) for authentication. All protected API endpoints require a valid authenticated session to access resources.

<Note>
The authentication implementation automatically handles session resolution from cookies (web) or headers (mobile).
</Note>

## Authentication Methods

Jottar supports two authentication methods depending on your platform:

### Cookie-Based (Web)

For web applications, authentication cookies are automatically sent with every request:

```typescript
const res = await fetch("https://jottar.vercel.com/api/me", {
  credentials: "include", // Automatically includes cookies
});
```

<Info>
Web applications don't need to manually manage authentication headers - the browser automatically includes session cookies with each request.
</Info>

### Header-Based (React Native)

For React Native and mobile applications, you need to explicitly pass authentication headers:

```typescript
const authToken = await getStoredAuthToken(); // Your token storage method

const res = await fetch("https://jottar.vercel.com/api/me", {
  headers: {
    "Content-Type": "application/json",
    "Authorization": `Bearer ${authToken}`,
    // Or use the same cookie header format that Better Auth provides
  },
});
```

<Warning>
React Native apps cannot use `credentials: "include"` as this is browser-only. You must explicitly manage and send authentication headers with each request.
</Warning>

## Getting Authenticated

To authenticate users, use the Better Auth endpoints:

### Sign Up

```typescript
POST /api/auth/sign-up
```

### Sign In

```typescript
POST /api/auth/sign-in
```

### Sign Out

```typescript
POST /api/auth/sign-out
```

<Note>
These authentication endpoints are handled by Better Auth's catch-all route at `/api/auth/[...all]`. Refer to the [Better Auth documentation](https://www.better-auth.com/docs) for detailed authentication flow examples.
</Note>

## Verifying Authentication

Use the `/api/me` endpoint to verify that a user is authenticated:

<CodeGroup>

```typescript Web
const res = await fetch("https://jottar.vercel.com/api/me", {
  credentials: "include",
});

const result = await res.json();

if (result.ok) {
  console.log("Authenticated as:", result.data.email);
} else {
  console.log("Not authenticated");
}
```

```typescript React Native
const res = await fetch("https://jottar.vercel.com/api/me", {
  headers: {
    "Authorization": `Bearer ${authToken}`,
  },
});

const result = await res.json();

if (result.ok) {
  console.log("Authenticated as:", result.data.email);
} else {
  console.log("Not authenticated");
}
```

</CodeGroup>

### Success Response

```json
{
  "ok": true,
  "data": {
    "id": "user-uuid",
    "email": "user@example.com",
    "name": "John Doe"
    // ... other user fields
  }
}
```

### Unauthenticated Response

```json
{
  "ok": false,
  "error": "Not authenticated"
}
```

## Handling 401 Errors

When a protected endpoint returns a 401 error, your application should:

1. Redirect the user to the login page (web)
2. Clear stored credentials (mobile)
3. Prompt the user to re-authenticate

### Example Error Handler

```typescript
type ApiResponse<T> = { ok: true } & T | { ok: false; error: string };

async function api<T>(
  path: string,
  init?: RequestInit,
): Promise<ApiResponse<T>> {
  const res = await fetch(`https://jottar.vercel.com${path}`, {
    ...init,
    headers: {
      "Content-Type": "application/json",
      ...(init?.headers ?? {}),
    },
  });

  // Handle 401 - Not Authenticated
  if (res.status === 401) {
    // Clear session and redirect to login
    await handleAuthFailure();
    return { ok: false, error: "Not authenticated" };
  }

  return res.json();
}

async function handleAuthFailure() {
  // Clear stored auth data
  await clearAuthToken();
  
  // Redirect to login (web)
  if (typeof window !== "undefined") {
    window.location.href = "/login";
  }
  
  // Or navigate to login screen (React Native)
  // navigation.navigate('Login');
}
```

## React Native Complete Example

Here's a complete example of setting up authentication in React Native:

```typescript
import AsyncStorage from '@react-native-async-storage/async-storage';

const API_BASE = "https://jottar.vercel.com";
const AUTH_TOKEN_KEY = "jottar_auth_token";

// Store auth token
export async function storeAuthToken(token: string) {
  await AsyncStorage.setItem(AUTH_TOKEN_KEY, token);
}

// Get stored token
export async function getAuthToken(): Promise<string | null> {
  return AsyncStorage.getItem(AUTH_TOKEN_KEY);
}

// Clear token
export async function clearAuthToken() {
  await AsyncStorage.removeItem(AUTH_TOKEN_KEY);
}

// API helper with auth
export async function authenticatedRequest<T>(
  path: string,
  init?: RequestInit,
): Promise<{ ok: true } & T | { ok: false; error: string }> {
  const token = await getAuthToken();
  
  const res = await fetch(`${API_BASE}${path}`, {
    ...init,
    headers: {
      "Content-Type": "application/json",
      ...(token ? { "Authorization": `Bearer ${token}` } : {}),
      ...(init?.headers ?? {}),
    },
  });

  if (res.status === 401) {
    await clearAuthToken();
    return { ok: false, error: "Not authenticated" };
  }

  return res.json();
}

// Usage
const notes = await authenticatedRequest<{ data: unknown[] }>("/api/notes");

if (!notes.ok) {
  console.error("Error:", notes.error);
  // Navigate to login screen
} else {
  console.log("Notes:", notes.data);
}
```

## Security Best Practices

<Warning>
**Never expose authentication tokens in your code or version control.**
</Warning>

1. Store tokens securely using platform-specific secure storage:
   - Web: `httpOnly` cookies (handled automatically by Better Auth)
   - React Native: [react-native-keychain](https://github.com/oblador/react-native-keychain) or `expo-secure-store`

2. Always use HTTPS in production

3. Implement token refresh logic to handle expired sessions

4. Clear tokens on sign-out and handle 401 responses globally
