## Overview

Jottar is designed to work seamlessly with React Native mobile apps through its REST API. This guide covers authentication, API usage, common workflows, and best practices for building a mobile client.

## Prerequisites

Before integrating Jottar with React Native:

<Steps>
  <Step title="Understand the REST API">
    Jottar exposes a full REST API under `/api/*` for managing notes, folders, tags, and user data.
  </Step>

  <Step title="Review the API types">
    Jottar provides a comprehensive TypeScript type reference in `API_TYPES_FOR_RN.md` that you can copy directly into your React Native project.
  </Step>

  <Step title="Set up authentication">
    Jottar uses Better Auth for authentication. You'll need to handle session tokens in your mobile app.
  </Step>

  <Step title="Configure trusted origins">
    Ensure your mobile app's origin is added to the trusted origins list in the Better Auth configuration.
  </Step>
</Steps>

## API Authentication from Mobile

### Better Auth Overview

Jottar uses **Better Auth** for authentication:

- All API requests require authentication
- Authentication is handled via session tokens
- Tokens are passed in cookies or headers
- The session is validated on every API request

### Authentication Flow

<Steps>
  <Step title="User signs in">
    The user signs in through the Better Auth flow (email/password, OAuth, etc.).
  </Step>

  <Step title="Receive session token">
    Better Auth returns a session token that identifies the user.
  </Step>

  <Step title="Store token securely">
    Store the session token securely in React Native using:
    - `@react-native-async-storage/async-storage` for persistence
    - Expo SecureStore for sensitive data
    - Keychain/Keystore for maximum security
  </Step>

  <Step title="Include token in requests">
    Include the session token in every API request via the `Authorization` header or cookies.
  </Step>
</Steps>

**Implementation Example:**

```tsx
import AsyncStorage from '@react-native-async-storage/async-storage';

// Store session token after login
await AsyncStorage.setItem('sessionToken', token);

// Retrieve token for API calls
const token = await AsyncStorage.getItem('sessionToken');

// Make authenticated request
const response = await fetch('https://your-jottar-instance.com/api/notes', {
  method: 'GET',
  headers: {
    'Authorization': `Bearer ${token}`,
    'Content-Type': 'application/json'
  }
});
```

### Authentication Utility

Every API route uses `getRequestUser()` to validate the session:

```tsx
// From ~/workspace/source/src/app/api/_utils.ts
export async function getRequestUser(req: Request) {
  const session = await auth.api.getSession({ headers: req.headers });
  if (!session?.user) return null;
  return session.user;
}
```

<Warning>
  All API requests return a 401 error if the session is invalid or missing. Make sure to handle authentication errors gracefully in your mobile app.
</Warning>

## Common Mobile Workflows

### Fetching Notes

**Get all notes:**

```tsx
import { ApiResponse, GetNotesResponse } from './api-types';

async function fetchNotes(): Promise<GetNotesResponse> {
  const token = await AsyncStorage.getItem('sessionToken');
  
  const response = await fetch('https://your-jottar-instance.com/api/notes', {
    method: 'GET',
    headers: {
      'Authorization': `Bearer ${token}`,
      'Content-Type': 'application/json'
    }
  });

  return response.json();
}
```

## Trusted Origins Configuration

### Why Trusted Origins Matter

Better Auth validates the origin of API requests to prevent CSRF attacks. You need to add your mobile app's origin to the trusted origins list.

### Mobile App Origins

For React Native apps, the origin depends on your setup:

- **Expo Go:** `exp://` or `http://localhost:19000`
- **Custom native app:** Your app's bundle identifier (e.g., `com.yourcompany.jottar`)
- **Web view:** The domain of your embedded web content

### Configuring Better Auth

Add your mobile app's origin to the Better Auth configuration:

```tsx
// In your Better Auth setup
import { betterAuth } from "better-auth";

export const auth = betterAuth({
  // ...
  trustedOrigins: [
    "https://your-jottar-instance.com",
    "exp://",                          // Expo Go
    "http://localhost:19000",          // Expo dev server
    "com.yourcompany.jottar"           // Native app
  ]
});
```

<Warning>
  If your mobile app's origin is not in the trusted origins list, API requests will be rejected with a CORS error.
</Warning>

## Handling Tiptap Content in Mobile

### Content Format

Jottar stores note content as **JSON** from the Tiptap editor:

```json
{
  "type": "doc",
  "content": [
    {
      "type": "heading",
      "attrs": { "level": 1 },
      "content": [
        { "type": "text", "text": "My Note Title" }
      ]
    },
    {
      "type": "paragraph",
      "content": [
        { "type": "text", "text": "This is a paragraph." }
      ]
    }
  ]
}
```

### Rendering Options

**Option 1: Use Tiptap in React Native**

If you want full rich-text editing in your mobile app, use a React Native Tiptap wrapper like:

- `@tiptap/react` (if using React Native Web)
- A custom WebView-based editor

**Option 2: Render as plain text**

For simpler read-only views, extract plain text from the JSON:

```tsx
function extractTextFromDoc(doc: any): string {
  if (!doc?.content) return "";

  let text = "";

  const walk = (node: any) => {
    if (node.type === "text") {
      text += node.text + " ";
    }
    if (node.content) {
      node.content.forEach(walk);
    }
  };

  doc.content.forEach(walk);
  return text.trim();
}
```

**Option 3: Render as Markdown**

Convert the Tiptap JSON to Markdown for display:

- Use a library like `tiptap-markdown` to convert JSON to Markdown
- Render Markdown in React Native using `react-native-markdown-display`

<Tip>
  For the best mobile experience, consider using a lightweight plain-text editor and storing notes as plain text or Markdown. This avoids the complexity of managing Tiptap JSON in React Native.
</Tip>

## Error Handling

### API Error Format

All API errors follow a consistent format:

```json
{
  "ok": false,
  "error": "Error message here"
}
```

### Common Errors

| Status Code | Error | Cause |
|-------------|-------|-------|
| 401 | Not authenticated | Missing or invalid session token |
| 404 | Not found | Note, folder, or tag doesn't exist or doesn't belong to user |
| 400 | Invalid request | Missing required fields or invalid JSON |
| 500 | Server error | Unexpected server-side error |

### Error Handling Example

```tsx
async function safeApiCall<T>(fn: () => Promise<T>): Promise<T | null> {
  try {
    const response = await fn();
    if ('ok' in response && !response.ok) {
      console.error('API error:', response.error);
      // Show error toast to user
      return null;
    }
    return response;
  } catch (error) {
    console.error('Network error:', error);
    // Show network error toast to user
    return null;
  }
}

// Usage
const notes = await safeApiCall(() => fetchNotes());
if (!notes) {
  // Handle error
  return;
}
```

## Best Practices

<Steps>
  <Step title="Cache data locally">
    Store fetched notes, folders, and tags in local state or a database like SQLite for offline access.
  </Step>

  <Step title="Implement optimistic updates">
    Update the UI immediately and rollback if the API call fails (same pattern as the web app).
  </Step>

  <Step title="Handle token expiration">
    Check for 401 errors and prompt the user to re-authenticate if their session expires.
  </Step>

  <Step title="Use pagination for large lists">
    While the API doesn't currently support pagination, consider implementing client-side pagination for performance.
  </Step>

  <Step title="Debounce content updates">
    When the user is typing, debounce API calls to avoid excessive requests.
  </Step>

  <Step title="Sync in the background">
    Use background tasks to sync notes, folders, and tags periodically.
  </Step>
</Steps>
